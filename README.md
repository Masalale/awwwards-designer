# Awwwards Designer — Skill Suite

A system of five coordinated AI agent skills for building award-quality websites from brand brief to shipped product. Design is generated by [Google Stitch MCP](https://stitch.withgoogle.com). Animation is derived from brand metaphor — not borrowed from templates.

Targets Awwwards **SOTD**, **SOTM**, and **SOTY**.

---

## How it works

```
Brief Interview → Invention Gate → Stitch MCP → Enhancement Layer → Quality Gate
```

| Phase | What happens |
|-------|-------------|
| **Brief Interview** | 7 questions. Brand name, identity, audience, animation tier, pages, interactive sections, references. |
| **Invention Gate** | Brand metaphor extracted → motion language defined → animation map produced → Stitch prompt written. No code until this gate clears. |
| **Stitch MCP** | Google Stitch generates all screens, one per route. A cross-page harmonisation pass enforces visual consistency. `build_site` outputs HTML + Tailwind CSS. `extract_design_context` produces `DESIGN.md` — the single source of truth for all design decisions. |
| **Enhancement Layer** | TanStack Start is scaffolded, Stitch HTML is integrated, then the animation and interaction layer is added. Library selection is driven by the animation map tier. `DESIGN.md` is the authority — nothing overrides it. |
| **Quality Gate** | 8-gate pass/fail checklist: Design · Creativity · Content · Usability · Performance · Animation · Technical · Pre-Launch. Ship only when every gate passes. |

The **Invention Gate** is what separates this from a template generator. Every animation decision traces back to a brand metaphor derived from the brief — not from a library of pre-built effects.

## Skills

| Skill | Role |
|-------|------|
| [`awwwards-designer`](./awwwards-designer/) | Orchestrator. Drives all 5 phases. |
| [`awwwards-animation-standards`](./awwwards-animation-standards/) | Easing vocabulary, animation complexity matrix, GPU animation rules, scroll, text, and transition patterns. |
| [`awwwards-performance-standards`](./awwwards-performance-standards/) | Core Web Vitals targets (LCP < 1.8s, CLS < 0.05, INP < 100ms), JS bundle budgets, LCP/CLS/INP optimisation techniques. |
| [`awwwards-quality-gate`](./awwwards-quality-gate/) | Pre-launch 8-gate checklist: Visual · Motion · Responsive · Accessibility · Performance · Interaction · Technical. |
| [`awwwards-visual-standards`](./awwwards-visual-standards/) | Typography, color token architecture, Lucide icons, spacing, layout grid. Deferred to `DESIGN.md` after Phase 3. |

The orchestrator (`awwwards-designer`) drives the workflow. The four standards skills are autonomous knowledge bases — the agent selects them based on what the current phase requires.

## Install

**Full suite:**

```bash
git clone https://github.com/Masalale/awwwards-designer.git
cd awwwards-designer
bash install.sh
```

Copies all five skills to `~/.agents/skills/` where Claude Code and compatible agents discover them automatically.

**Individual skill via the skills CLI:**

```bash
bunx skills add https://github.com/Masalale/awwwards-designer --skill awwwards-designer
```

## Animation tiers

The tier is set during the Brief Interview and determines the animation ceiling and JS bundle budget.

| Tier | Capability | JS budget (gzip) |
|------|-----------|-----------------|
| **1** | CSS-only animations and transitions | < 20 KB |
| **2** | Scroll-triggered reveals, smooth scroll, animated text, page transitions | < 120 KB |
| **2.5** | Everything in Tier 2 + three-dimensional spatial elements in specific zones | < 150 KB |
| **3** | Everything in Tier 2 + full GPU-rendered scenes, generative art, GLSL shaders | < 300 KB |

> [!NOTE]
> Build from the tier up. If the animation map does not demand Tier 3, do not implement it. Every tier has Awwwards-winning examples — complexity is not a proxy for quality.

## Tech stack

| Concern | Default | Status |
|---------|---------|--------|
| Framework | TanStack Start + React | Mandatory |
| Package manager | Bun | Mandatory |
| Icons | Lucide | Mandatory |
| Smooth scroll | Lenis (`smoothTouch: false`) | Preferred |
| Scroll orchestration | GSAP + ScrollTrigger | Tier 2+ |
| Component animation | Framer Motion | State-driven mount/unmount |
| Text splitting | SplitType | Character/word animation |
| WebGL | Three.js / React Three Fiber | Tier 2.5+ |
| Routing | TanStack Router | Built into TanStack Start |

Library selection is capability-first — the animation map defines what capability is needed, and the agent selects the best tool for that capability. The defaults above are the proven production choices; deviating requires documented rationale.

## Hard stops

These are non-negotiable. Any violation is a build failure.

- **No code before the Invention Gate completes** — The animation map must exist before any screen is generated or any code is written
- **TanStack Start only** — No Astro, Next.js, or any other framework
- **Bun only** — No npm, npx, or yarn
- **Lucide icons only** — One system, one visual language
- **`100dvh` not `100vh`** — Accounts for mobile address bar resize on full-screen sections
- **No touch scroll override** — Native touch momentum is sacred; Lenis must run with `smoothTouch: false`
- **Descender safety on display text > 48px** — `overflow: visible` + `padding-bottom: 0.18em` minimum; measure per font via Canvas API
- **Context7 before any library integration** — Training data is stale; fetch current API docs first

## Performance targets

All Awwwards-tier targets — stricter than the web.dev "Good" thresholds.

| Metric | Target |
|--------|--------|
| LCP | < 1.8s |
| CLS | < 0.05 |
| INP | < 100ms |
| FCP | < 1.2s |
| Lighthouse Performance | ≥ 90 |

WebGL and 3D scenes must initialise after first paint — never before LCP.

## Quality gate

Eight gates. All must pass before shipping.

| Gate | Area |
|------|------|
| 1 | Design fidelity to `DESIGN.md` |
| 2 | Creativity — unique concept, signature interaction, memorable first impression |
| 3 | Content — no lorem ipsum, clear hierarchy, specific CTAs |
| 4 | Usability — mobile 375px, keyboard nav, 44px touch targets, forms |
| 5 | Performance — LCP ≤ 1.8s, CLS ≤ 0.05, INP ≤ 100ms |
| 6 | Animation — purpose-driven, physical easing, reduced-motion support |
| 7 | Technical — clean build, no dead routes, descender safety, no `transition: all` |
| 8 | Pre-launch audit sequence — Lighthouse, mobile, keyboard, asset check |

> [!WARNING]
> A site that fails any gate is not ready for Awwwards submission. Fix before shipping — no exceptions.
